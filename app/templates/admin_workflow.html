Admin-Only System Workflow

This workflow is private for the owner/admin only.

Public users can use the dashboard and suggestion page, but this internal architecture flow stays protected to reduce idea copying.

01Search Mode

Keyword only, keyword + location, or static demo mode.

02Scraper Layer

Playwright opens pages and BeautifulSoup parses job data.

03MongoDB Storage

Duplicate-safe job storage with timestamp fields and indexes.

04Analytics Engine

Pandas and MongoDB aggregations generate KPI-ready insights.

05Dashboard

Chart.js visualizes real-time and historical hiring intelligence.

Automation Workflow

  1. APScheduler triggers daily scraping at the configured hour.
  2. Scraping logs are written before and after each job.
  3. Daily snapshots calculate job count, skill demand, hiring companies, and remote ratio.
  4. Weekly snapshots compare growth and skill/company trends.
  5. Monthly snapshots prepare salary, technology, location, and historical reports.

Privacy Design

  1. Workflow page is protected by admin login.
  2. Public users only see dashboard, jobs, analytics, logs, export, and suggestion page.
  3. Admin password is configured from .env, not hardcoded for production.
  4. Suggestions are stored in MongoDB and optionally sent to the owner email.
